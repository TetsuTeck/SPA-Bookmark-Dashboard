# Bookmark Dashboard 🌐

Markdown（マークダウン）を記述するだけで、自分専用の美しく機能的なWebブックマークポータルを自動生成する、1ファイル完結型のダッシュボードツールです。
データベースサーバーを介さず、登録したリンクや各種お気に入り設定はすべてあなたのブラウザ（LocalStorage）の中に安全に自動保存されます。

---

## ✨ 主な特徴 (Features)

* **1ファイル完結 (Single Page Application)**: サーバー不要、HTMLファイルをブラウザで開くだけで爆速動作。
* **ゼロ依存ローカルサーバー (Zero Dependency)**: スクリーンショット撮影用のNode.jsサーバーは標準ライブラリのみで構築されており、`npm install` などのパッケージ導入が一切不要です。
* **ローカル自動スクリーンショット保存**: 外部のクラウドサービスに依存することなく、ローカルのヘッドレスChromeを利用してリンク先スクリーンショットを自動生成し、フォルダに保存・キャッシュ可能。
* **【新機能】サーバー停止時・オフライン時のスクショ永続表示**:
  - `server.js` を停止した後や、ダッシュボードHTMLを直接 `file:///` プロトコル等でブラウザで開いた場合でも、ローカルフォルダ (`screenshots/`) に保存済みのスクリーンショット画像をフロントエンド側で直接読み込み、消えずに永続表示できます。
* **2段式超ワイド検索バー**: 画面幅100%をフル活用し、タイトル・URL・備考をリアルタイムに横断検索。
* **ハイブリッド・マルチレイアウト**: 詳細な情報を一覧できる「テーブル表示」と、最大10列まで可変可能な「タイル表示」を1タップ切り替え。
* **個別画像＆フォールバック対応**: タイルごとに任意の画像（Web URL / ローカル相対パス）をセット可能。画像読み込みエラー時も標準アイコンへ安全に自動フォールバック。
* **動的ソート機能**: 「記述順」「名前順（A-Z）」「クリック回数順」の3つのソートモードを搭載。
* **160種超アイコンピッカー**: カタログからアイコンを選択するだけで、裏側のマークダウンコードを自動書き換え。
* **Canvas背景圧縮アップロード**: お気に入りの画像を背景に設定可能（ブラウザ制限を回避する自動軽量化付き）。
* **ローカルファースト＆セキュア**: 外部へのデータ送信は一切なし。完全プライベートな個人専用仕様。

## 📥 インストール方法 (Installation)

本ツールはローカルファーストかつ依存パッケージゼロで設計されているため、セットアップは非常に簡単です。

### 前提要件 (Prerequisites)
1. **Node.js** (v14以上推奨) - スクリーンショット自動取得機能を使用する場合にのみ必要です。
2. **Google Chrome** - ヘッドレスモードでの撮影に使用します。
   - ※デフォルトではWindowsの標準インストールパス (`"C:\Program Files\Google\Chrome\Application\chrome.exe"`) を参照します。別のOSやカスタムパスを使用する場合は、`server.js` 内の `CHROME_PATH` (11行目) をお使いの環境に合わせて書き換えてください。

### セットアップ手順
1. 本リポジトリの全ファイル（`spa_bookmark_dashboard.html`, `server.js`）を任意のローカルフォルダにダウンロードまたはクローンします。
2. **インストール作業は不要です！** `npm install` 等を実行することなく、そのまますぐに起動できます。

## 📸 自動スクリーンショット機能 (Auto-Screenshot)

画像が設定されていないブックマークについて、リンク先ホームページのスクリーンショットをローカル環境で自動取得し、キャッシュ画像としてタイル表示できます。
ハッシュ値（MD5）を用いて画像ファイルを管理するため、表示が非常に高速です。

### 仕組みと動作フロー
1. 各ブックマークのURLからクライアント側で一意の MD5 ハッシュ値を生成します。
2. `screenshots/<hash>.png` を優先的に読み込みます。
3. すでにスクリーンショット画像が存在する場合、**サーバーが停止していても**画像は表示され続けます。
4. 画像が未取得（ローカルに存在しない）で「スクショ自動取得」がONの場合、エラーを検知して自動的にローカルのサーバーAPI (`http://localhost:3000/api/screenshot`) へリクエストを送り、ヘッドレスChromeを起動して自動撮影します。

## 🚀 使い方 (How to Use)

### 1. アプリの起動方法

#### A. ローカルサーバー経由（スクリーンショット機能対応・推奨）
1. ターミナル（コマンドプロンプトやPowerShell等）で本ディレクトリを開き、以下を実行して起動します：
   ```bash
   node server.js
   ```
2. ブラウザで `http://localhost:3000` を開きます。
3. ダッシュボード上の「スクショ自動取得」トグルをONにします。

#### B. スタンドアロン動作（静的ファイル単体）
1. 本フォルダにある `spa_bookmark_dashboard.html` をブラウザにドラッグ＆ドロップするか、ダブルクリックして直接開きます（URLが `file:///...` で始まります）。
2. ※このモードでも、すでにローカルフォルダ `screenshots/` 内に存在するスクリーンショット画像はそのまま表示されます。ただし、未撮影のブックマークの新規自動取得は行われません。

---

### 2. ブックマークの編集・登録

1. 画面右上にある **「エディタを開く」** ボタンをクリックします。
2. 画面全体に広がるエディタ領域に、以下の文法ルールに従ってブックマークを自由に記述します（Markdown形式）。
3. 記述が終わったら **「編集を終了してダッシュボードを見る」** をクリックすると、自動保存されて即座に反映されます。

#### 📝 マークダウンの記述ルール

```markdown
# 🌐 カテゴリ名（シャープの後ろにスペースを入れてカテゴリを定義。自動でタブ化されます）
- [リンクタイトル](URL) {icon: アイコン名} {img: 画像パス} : 備考（ここにメモや説明を記述）

# 💻 開発ツール
- [GitHub](https://github.com) {icon: github} : コード管理プラットフォーム。
- [Google](https://google.com) {icon: google} {img: https://images.unsplash.com/photo-1579546929518-9e396f3cc809?w=300} : 任意のロゴ写真をタイル画像に指定した例。
- [ローカルサイト](http://localhost:8080) {img: ./my-icon.png} : 同一フォルダ内のローカル画像を指定する例。
```

---

### 3. 便利な機能と操作

* **手動スクショ更新**: 
  画像にカーソルを合わせると表示される左上の「更新」マーク（またはリストビューの更新ボタン）をクリックすれば、いつでも個別に最新のキャプチャをオンデマンドで再撮影できます。
* **背景画像のカスタム設定**: 
  ヘッダー右端の背景選択サークルから、プリセット背景への切り替えのほか、`＋`ボタンから手持ちの画像を背景としてアップロード（クライアント側で軽量化されてLocalStorageに保存）できます。
* **ダークモード/ライトモード**: 
  ヘッダーの太陽/月アイコンをクリックして、いつでもワンタップで切り替え可能です。
